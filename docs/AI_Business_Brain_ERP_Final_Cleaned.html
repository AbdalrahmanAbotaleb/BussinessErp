System Architecture

The AI Business Brain employs a robust Distributed 3-Tier Architecture. This separation allows for component-level scaling and simplifies the integration of future cross-platform clients (e.g., Mobile/Web).

Architectural Diagram

The system implements a granular **Role-Based Access Control** system. Roles (Admin, Manager, Employee) govern UI visibility and BLL execution permissions.

Sales are the lifeblood of the system. To prevent inventory fragmentation, all sales are processed using **Database-Level Atomic Transactions**. A sale is only committed if the stock is successfully decremented and the financial record is created.

Security Architecture

The AI Business Brain follows a "Secure by Default" philosophy.

1. Cryptographic Hashing

All passwords undergo advanced processing via `SecurityHelper.cs`. The implementation utilizes SHA256 to create an irreversible fingerprint of the user's password. 

public static string HashPassword(string plainText)
{
    using (var sha256 = SHA256.Create())
    {
        byte[] bytes = sha256.ComputeHash(Encoding.UTF8.GetBytes(plainText));
        return BitConverter.ToString(bytes).Replace("-", "").ToLower();
    }
}

2. Admin Fail-Safe Logic

To prevent absolute system lockout (e.g., if all admin users are deleted), the application features a startup bootstrapper in `Program.cs`. It checks the database availability and ensures a default administrator exists before the login screen appears.

3. Data Protection Strategy

  • SQL Injection Prevention: Every repository call uses `SqlParameter`. No raw string concatenation is permitted in the Data Access Layer.

Example Technical Procedure Flow (sp_AddSale)

-- TRANSACT-SQL IMPLEMENTATION
BEGIN TRANSACTION;
    -- Header Creation
    INSERT INTO Sales (Date, Total) VALUES (GETDATE(), @Total);
    SET @SaleID = SCOPE_IDENTITY();

    -- Item Propagation (JSON Parsing)
    INSERT INTO SaleItems (SaleId, ProductId, Qty)
    SELECT @SaleID, ProductId, Quantity FROM OPENJSON(@Items);

    -- Automatic Stock Sync
    UPDATE Products SET Quantity = Quantity - @Qty WHERE ID = @PID;
COMMIT TRANSACTION;

This flow ensures that if the server crashes mid-execution, the transaction is rolled back, leaving the database in a consistent state.

Performance Optimization

1. Async-First Architecture

Every database request in the ERP is Asynchronous. We use `.ExecuteReaderAsync()` and `.ExecuteScalarAsync()` to release the UI thread while waiting for SQL response. This keeps the application "Fluid" and prevents "Not Responding" windows during heavy report generation.

2. Repository Pattern & Connection Pooling

The `DatabaseHelper` manages a centralized connection pool. Instead of opening/closing thousands of connections, we reuse them efficiently, reducing the overhead of the TCP handshake with SQL Server.

3. SQL Optimization

  • Non-Clustered Indexes: Applied to `Date` and `Name` columns for 10x faster range queries.
  • JSON Batching: The system sends bulk items as JSON strings to Stored Procedures, reducing network round-trips from the Client to the Database.

Deployment Guide

The AI Business Brain is designed for single-click environment parity.

1. Database Setup

  1. Install SQL Server (Express or Standard).
  2. Execute the `SQL/DatabaseSetup.sql` script to create schema, SPs, and Triggers.
  3. Execute the `SQL/SeedData.sql` script to populate initial categories and default users.

2. Configuration

Update the connection string in `App.config` to point to the production server.

<connectionStrings>
  <add name="Default" connectionString="Server=PROD_SRV;Database=BussinessDB;..." />
</connectionStrings>

3. Production readiness

  • Check that the `Logs/` directory is writable by the application.
  • Verify that `.NET Framework 4.8` is installed on target workstations.
